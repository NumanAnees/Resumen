Introduction

First of all, I would like to thank you for your purchase of React Resume Builder. in this documentation, I will show you how to work with this web app. Don't worry if you are not a developer this web app is easy to work with. and if you are a developer there is a section in this documentation where you will find all the necessary information you need to be able to deploy with no difficulties.

Installation

Step One : Firebase

Go to your firebase account and create new project

Give your project a name

Disable analytics since we are using in app statistics

Click on this icon to let firebase know that we want to use firebase for web

Give your app a name and uncheck firebase hosting since we will use a vps.

We need to connect our app with firebase and this needs to be done from /Resumen/.env file. we need to replace the data from firebase into ".env" file. Copy the values from firebase into .env file.

We are going to need the built-in firebase authentification service. Please activate Email Authentication, Google Authentication.

We need to change the security rules in firestore. with the one included with the project ( securityRules.txt)

Add your domain name to authorized domains. this will let our app login successfully using social authentification

Step Three : Deployment

In order to deploy our application we will need an ubuntu vps with Nginx from digitalocean. By using our referal link you will get 100$ on your balance that will be enough to run the app for a long period of time. Create Digitalocean account

Click on create droplet. then click on marketplace and pick LEMP

Pick a location close to you

Set your root password

Then click on Create droplet

Add domain name

To add a domain name follow this tutorial right here How to add domain name to Digitalocean

once you add Digitalocean Namespaces to your domain provider now you can add the domain

Enter your domain without www. and click on add domain

Now we have to add a new host name, type www and select your droplet and click create record

Let's wait couple of minutes then navigate to your domain you should see this Digitalocan default page

If you added your domain but it doesnt show you this page. this means you have done something wrong. and it is advised to check if you added namespaces to you domain provider corectly.

As you can see that our website is being served over http. which is not secure. next we will configure our vps to serve our website throu HTTPS

Connect to our vps

We need to connect to our vps with SSh for this reason we are goin to use Putty. You can download Putty from Here add your server ip address and click open. it will ask you to authenticate with root.

Once you authenticate run this command

sudo apt-get update

Then this one

sudo add-apt-repository ppa:certbot/certbot

it will ask you to click enter, once its finished run this one

sudo apt install python3-certbot-nginx

then this one to open our Nginx configuration file

sudo nano /etc/nginx/sites-available/digitalocean

and do this changes

Change this ---> server_name localhost;
with this ---> server_name yourdomain.com www.yourdomain.com;

Change this ---> try_files $uri $uri/ =404;
with this ---> try_files $uri /index.html;

To save and exit file

Ctrl+X yes Enter

Then run all this commands one by one

sudo ufw allow 'Nginx Full'

sudo ufw allow 'Nginx HTTP'

sudo ufw allow 'Nginx HTTPS'

sudo ufw allow https

Add your domain to the next command

sudo certbot --nginx -d domainName.com -d www.domainName.com

Will ask you to add your email

Then it will ask you to agree on their terms of service

Will ask you if you want to redirect all traffic to HTTPS we pick number 2.

Now if you navigate to your domain. you will see that the connection is secure

In you vps command line send this command

cd /var/www/html

this commande to remove the default page digitalocean showing us when we navigate to our domain

rm index.html

now in you computer go to /Resumen/build file and send this command, (add your server ip address before)

scp -r ./* root@SERVER IP HERE :/var/www/html

will ask you to add your password then the upload will start

And that would be it for deployment. remember if you have any difficulties try the tutorial video. if still you have issues open a support ticket and we will be happy to assist you

Backed configuration

1 - Our backend will handle resume building. export to pdf. and payments processing. connect to your vps using Putty. and lets install Node.js with this command

sudo apt install nodejs

It will ask you to click yes.

2 - Then this command will a dependency needed

sudo apt install chromium-browser

3 - And this one too

sudo apt install npm

4 - Our backend will use the port 8080, for this reason we need to make sure to allow 8080 traffic

sudo ufw allow 8080

5 - Enter this command to navigate to /var/www directory

cd /var/www

6 - Enter this command to create a new directory where our backend files will be located

mkdir backend

7 - Enter this command to navigate to the backend directory

cd backend

8 - Now in your computer open Resumen/backend and open the command prompt in this directory

Resumen_V0.1.6\Backend

9- Open index.js with notepad and add your domain in this lines

const websiteName = "domain.com" /// domain.com
key: fs.readFileSync('/etc/letsencrypt/live/domain.com/privkey.pem'),
cert: fs.readFileSync('/etc/letsencrypt/live/domain.com/fullchain.pem'),

10- Then save and run this command to upload files to our vps (make sure you add your vps ip address)

scp -r ./* root@ServerIp:/var/www/backend

11- Now go back to your server Putty window and make sure you are in the backend directory

cd /var/www/backend

12 - We need to create a file that wil hold our stripe key and to do this enter this command

sudo nano .env

13- add this code , (Replace stripe secret key with yours)

STRIPE_SECRET="sk_test_51IZbSFKcjLOayGZuRkLjy61a6X1646KNLu1eH7BR2lDLHFIYNxMzerJm8QxDjngOydrwouhsmNtKFPlS0LnVnp2992300SRk7mS8D"

14- Make sure you are in var/www/backend and run this command

npm install

15 - Now our backend is installed we need to configure Nginx to receive the api calls to the backend

sudo nano /etc/nginx/sites-available/digitalocean

16 - under the server_name add the following location part: (to save the file Ctrl+x yes and Enter)

location /api { proxy_pass https://localhost:8080;
proxy_http_version 1.1;
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection 'upgrade';
proxy_set_header Host $host;
proxy_cache_bypass $http_upgrade;
}

17 - Again make sure you are in var/www/backend and run this commands one by one

sudo npm install -g pm2
pm2 start index.js
pm2 startup systemd

Then we run this command to restart Nginx, and that would be it.

sudo service nginx restart

Google Authentication not working

For this problem the problem will most likely be not authorizing your domain in firebase,

1. Open Firebase Account Click on Authentication, then Sign-in Method

2. Scroll down and make sure you added your domain to Authorized Domains
